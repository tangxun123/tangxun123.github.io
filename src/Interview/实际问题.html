面试遇到的实际问题 ​

跨域的解决方式 ​

1. CORS（跨域资源共享） ​

服务端设置响应头，允许指定源的请求。例如，在响应头中添加：

  • Access-Control-Allow-Origin: * 允许所有域，或者指定具体域名。
  • Access-Control-Allow-Methods: GET, POST, PUT, DELETE 等允许的请求方法。
  • Access-Control-Allow-Headers: Content-Type, Authorization 等允许的请求头。

对于复杂请求（如Content-Type为application/json），浏览器会先发送一个预检请求（OPTIONS），服务器需要正确处理预检请求。

2. JSONP（JSON with Padding）： ​

利用<script>标签没有跨域限制的特性，通过动态创建script标签，将请求数据作为参数传递给服务器，服务器返回一段可执行的JavaScript代码（通常是一个函数调用，函数名由客户端指定，数据作为参数），从而实现跨域获取数据。

缺点：只支持GET请求，且需要服务器配合。

3. 代理服务器 -- 同源服务器、nginx反向代理： ​

在同源策略下，浏览器向同源服务器发送请求，然后由同源服务器代理请求目标服务器。因为服务器之间没有同源限制，所以可以解决跨域问题。

开发环境中，常用的工具如webpack-dev-server的proxy配置（内置 http-proxy-middleware 中间件）

nginx反向代理。

4. WebSocket： ​

WebSocket协议本身支持跨域，可以在建立连接时通过HTTP请求头进行跨域控制，服务器验证Origin头决定是否允许连接。

vue代理请求怎么做的 ​

Vue Proxy 代理的核心原理是利用 node开发服务器 作为中间转发层，通过 http-proxy-middleware 实现请求的转发

http-proxy-middleware 的核心原理是在本地开发服务器（如 webpack-dev-server）上设置一个代理层

  • 当你的前端应用在开发服务器上运行时，所有网络请求都会先发送到这个本地服务器。
  • 如果请求路径匹配了你设置的代理规则，那么本地服务器会代为转发这个请求到指定的目标服务器。
  • 这样做之所以能解决跨域问题，是因为浏览器有同源策略限制，但服务器之间没有。你的前端代码（运行在浏览器中）只与同源的本地开发服务器通信，而由本地服务器去请求不同源的远程资源
js
// vue.config.js
proxy: {
  '/api': {
    target: 'http://localhost:3000',
    pathRewrite: { '^/api': '' },
    changeOrigin: true
  }
}
js
// http-proxy-middleware 内部逻辑
function middleware(req, res, next) {
  if (shouldProxy(req.url)) {
    // 创建到目标服务器的请求
    const proxyReq = http.request(targetOptions, (proxyRes) => {
      // 将目标服务器的响应返回给浏览器
      res.writeHead(proxyRes.statusCode, proxyRes.headers);
      proxyRes.pipe(res);
    });
    
    // 转发原始请求
    req.pipe(proxyReq);
  } else {
    next();
  }
}

// 本地服务器 实际创建的代理中间件
const { createProxyMiddleware } = require('http-proxy-middleware');
app.use('/api', createProxyMiddleware({
  target: 'http://backend-server.com',
  changeOrigin: true,
  pathRewrite: {
    '^/api': '', // 移除路径中的 '/api' 前缀
  },
  onProxyReq: (proxyReq, req, res) => {
    console.log('代理请求:', req.url, '->', proxyReq.path);
  },
  onProxyRes: (proxyRes, req, res) => {
    // 可以在这里处理响应，例如添加自定义响应头
  }
}));

自定义指令 ​

1. 钩子函数 ​

一个指令定义对象可以提供如下几个钩子函数 (均为可选)：

  • bind：只调用一次，指令第一次绑定到元素时调用。在这里可以进行一次性的初始化设置。
  • inserted：被绑定元素插入父节点时调用 (仅保证父节点存在，但不一定已被插入文档中)。
  • update：所在组件的 VNode 更新时调用，但是可能发生在其子 VNode 更新之前。指令的值可能发生了改变，也可能没有。但是你可以通过比较更新前后的值来忽略不必要的模板更新 (详细的钩子函数参数见下)。
  • componentUpdated：指令所在组件的 VNode 及其子 VNode 全部更新后调用。
  • unbind：只调用一次，指令与元素解绑时调用。

2. 钩子函数参数 ​

  • el：指令所绑定的元素，可以用来直接操作 DOM。
  • binding：一个对象，包含以下 property：

    • name：指令名，不包括 v- 前缀。

    • value：指令的绑定值，例如：v-my-directive="1 + 1" 中，绑定值为 2。

    • oldValue：指令绑定的前一个值，仅在 update 和 componentUpdated 钩子中可用。无论值是否改变都可用。

    • expression：字符串形式的指令表达式。例如 v-my-directive="1 + 1" 中，表达式为 "1 + 1"。

    • arg：传给指令的参数，可选。例如 v-my-directive:foo中，参数为 "foo"。

    • modifiers：一个包含修饰符的对象。例如：v-my-directive.foo.bar 中，修饰符对象为 { foo: true, bar: true }。

  • vnode：Vue 编译生成的虚拟节点。
  • oldVnode：上一个虚拟节点，仅在 update 和 componentUpdated 钩子中可用。

除了 el 之外，其它参数都应该是只读的，切勿进行修改。如果需要在钩子之间共享数据，建议通过元素的 dataset 来进行。

3. 动态指令参数 ​

指令的参数可以是动态的。例如，在 v-mydirective:[argument]="value" 中，argument 参数可以根据组件实例数据进行更新！这使得自定义指令可以在应用中被灵活使用。通过 binding.arg 获取动态参数

    {{ title }} {{ count }}
</template>

切片上传 断点续传 ​

切片上传和断点续传的功能。前端主要思路如下：

  1. 文件切片：将大文件分割成多个小块（例如，每块1MB）关键技术 Blob.slice()。
  2. 文件哈希：计算文件的哈希值，用于标识文件，实现断点续传（同一文件即使多次上传也使用相同标识）SparkMD5 三方库。
  3. 上传切片：将切片逐个上传，并记录上传成功的切片 并发控制。
  4. 断点续传：在上传前，先询问服务器哪些切片已经上传，然后只上传未上传的切片；或者使用localStorage保存上传进度，实现断点续传。
  5. 合并切片：所有切片上传完成后，通知服务器合并切片。

后端接口实现。后端需要提供以下接口：

  1. 检查文件上传状态（返回已上传的切片索引）
  2. 上传切片
  3. 合并切片

关键代码 ​

  1. 文件切片与哈希计算
js
class FileUploader {
  constructor(options = {}) {
    this.chunkSize = options.chunkSize || 1024 * 1024; // 1MB
    this.concurrentUploads = options.concurrentUploads || 3;
    this.file = null;
    this.fileHash = '';
    this.totalChunks = 0;
    this.uploadedChunks = new Set();
    this.isUploading = false;
    this.isPaused = false;
    this.activeRequests = [];
  }

  // 计算文件哈希（简化版，实际使用SparkMD5）
  async calculateFileHash(file) {
    return new Promise((resolve) => {
      // 简化实现，实际应使用SparkMD5计算文件内容哈希
      const hash = `file_${Date.now()}_${Math.random().toString(36).substr(2)}`;
      resolve(hash);
    });
  }

  // 文件切片
  createFileChunks(file) {
    const chunks = [];
    let current = 0;
    
    while (current < file.size) {
      const chunk = file.slice(current, current + this.chunkSize);
      chunks.push({
        chunk,
        index: chunks.length,
        start: current,
        end: Math.min(current + this.chunkSize, file.size)
      });
      current += this.chunkSize;
    }
    
    return chunks;
  }
}

深度和广度优先遍历 ​

深度优先遍历（DFS） ​

js
function dfsRecursive(node) {
  if (!node) return;
  node.children.forEach(child => dfsRecursive(child));
}

广度优先遍历（BFS） ​

js
function bfsRecursive(root) {
    if (!root) return [];
    const queue = [root];
    const result = [];
    
    while (queue.length) {
        const node = queue.shift();
        result.push(node);
        // 将子节点按顺序加入队列
        for (let child of node.children) {
            queue.push(child);
        }
    }
    return result;
}
